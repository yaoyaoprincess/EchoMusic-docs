# 🧩 插件系统

EchoMusic 从 v2.2.6-beta.11 起支持完整的插件系统。插件可以扩展界面、添加功能、修改外观，让你打造专属的音乐播放体验。

## 什么是插件

插件是由社区成员或你自己编写的小型代码包，可以在 EchoMusic 中：

- 📌 注册独立页面或侧边栏入口
- 🎨 注入自定义 CSS 样式
- 🎵 监听播放器状态、控制播放
- 🖼️ 创建桌面浮窗（如灵动岛歌词）
- 📝 提供自定义歌词内容或歌词动效
- 🔍 接管特定歌曲的音源解析
- ⚙️ 添加设置面板
- 📋 在歌曲右键菜单中添加选项

插件定位接近 VS Code / Obsidian 的高自由度本地扩展：EchoMusic 提供宿主 API，插件自由发挥。

## 安装插件

### 在线插件源（推荐）

EchoMusic 内置了在线插件源，可以直接浏览和安装：

1. 打开 EchoMusic，进入「**插件管理**」
2. 点击「**在线插件**」标签页
3. 浏览可用插件列表
4. 点击「**安装**」按钮，插件会自动下载并安装

### 添加第三方插件源

你可以添加其他 GitHub 仓库作为插件源：

1. 在「插件管理」→「在线插件」中点击「**添加源**」
2. 输入插件源地址，例如：
   ```
   https://github.com/hoowhoami/EchoMusicPlugins
   ```
3. 刷新后即可看到该源提供的插件

### 手动安装

1. 在「插件管理」中点击「**打开目录**」
2. 将下载的插件文件夹（包含 `manifest.json`）复制进去
3. 返回 EchoMusic，插件会自动出现在列表中

```
<EchoMusic 插件目录>/
  hello-echo/
    manifest.json
    index.js
    style.css
```

## 管理插件

| 操作 | 说明 |
|------|------|
| **启用/禁用** | 点击插件卡片上的开关即可切换。禁用时会调用插件的 `deactivate` 清理注册的资源 |
| **卸载** | 点击删除按钮，会同时删除插件目录、清除数据和故障记录 |
| **设置** | 部分插件提供设置面板，点击齿轮图标进入配置 |
| **更新** | 在线安装的插件，插件管理会检查更新提示 |

## 安全模式

EchoMusic 提供全局「**插件安全模式**」，开启后不会加载任何插件，但保留每个插件的启用状态。

### 何时使用安全模式

- 某个插件导致界面卡死或崩溃
- 启动后窗口无响应
- 排查插件兼容性问题

### 进入安全模式

**方式一：插件管理页**

在「插件管理」中打开「插件安全模式」开关，然后重启应用。

**方式二：命令行启动**

```bash
EchoMusic --safe-mode
```

开发环境下：

```bash
pnpm exec electron . --safe-mode
```

### 自动触发

EchoMusic 会记录插件启动和运行状态。如果某个插件导致渲染进程异常退出，主进程会自动尝试切换到安全模式并重载窗口。

## 故障排查

如果遇到插件相关问题：

1. **启用安全模式**：先进入安全模式确认是插件导致的问题
2. **逐一排查**：关闭安全模式后逐个启用插件，找出问题插件
3. **查看异常记录**：插件管理页会用 ⚠️ 标记异常插件，点击可查看异常来源、时间、消息和堆栈
4. **清除记录**：如果问题已解决，可以清除异常记录

::: warning 注意
插件运行在渲染进程中，具有较高权限。请只安装来源可信的插件。如果插件来源不明，建议先阅读其代码再安装。
:::

## 命令行参数

| 参数 | 说明 |
|------|------|
| `--safe-mode` | 以安全模式启动，不加载任何插件 |
